아래는 RAPTOR v2.17.0 - 렌더링 파이프라인(Step 4 KIE API / Step 5 FFmpeg) 완전 분리 수술의 최종 변경 내용이다.
현재 변경된 상태를 실제 소스 코드(main.py, src/components/RaptorWorkflow.tsx)와 교차 검증하여 아키텍처 사후 리뷰(Post-Review) 보고서를 한국어로 작성해줘.

[이번 패치 배경]
다중 접속 환경에서 긴 시간 동안 결합되어 있던 렌더링 파이프라인이 커넥션 누수(Errno 11) 및 에러 파악의 모호함을 초래했다. 이에 따라 비디오 소스 생성(Step 4)과 최종 영상 조립(Step 5) 과정을 완전히 분리(Decoupling)하였다.

[검토 대상 변경 사항 요약]

변경 1: 백엔드 `main.py` 라우터 분리 및 KIE API 커넥션 정리
  - 기존의 `@app.post("/api/render-stream")`를 삭제하고 두 개로 분리함.
  - `@app.post("/api/generate-video-clips")`: KIE AI 비디오 생성 전용.
    - `httpx.AsyncClient()`를 명시적으로 생성하고 `try...finally` 블록에서 `await client.aclose()`를 호출하여 커넥션 누수 차단.
  - `@app.post("/api/render-final")`: FFmpeg 합성 전용.

변경 2: 프론트엔드 `src/components/RaptorWorkflow.tsx` 로직 및 UI 분리
  - 기존 `handleRenderVideo`를 `handleGenerateClips`와 `handleRenderFinal` 두 개의 독립적인 함수로 분리.
  - [비디오 클립 생성 (Step 4)] 버튼 신설: `handleGenerateClips` 호출.
  - [최종 영상 조립(렌더링) (Step 5)] 버튼 신설: 모든 비디오 클립이 준비(video_url 존재 또는 use_image_only)되었을 때만 활성화되며 `handleRenderFinal` 호출.
  - 신호등(🟢🟠🔴) 서버 상태 표시는 최종 영상 조립(Step 5) 버튼 영역에만 국한되도록 렌더링 변경.
