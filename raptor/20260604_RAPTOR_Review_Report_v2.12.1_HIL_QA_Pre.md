# RAPTOR v2.12.1 HIL QA 핫픽스 사전 리뷰 보고서 (Pre-Review Report)

본 보고서는 기획자의 실전 HIL 테스트 결과 반영을 위한 `v2.12.1` 실행 계획서(`implementation_plan.md`)에 대해 Claude Code의 관점에서 무결성과 상태 보호, 보안 정합성을 사전 검토(Pre-Review)한 결과물입니다.

---

## 1. 검토 의견 및 총평 (Overall Assessment)

제시된 `v2.12.1` 실행 계획서는 실전 HIL 테스트에서 적발된 인증 우회, 상태 누수, 폼 입력 보안 결함, UI 겹침 및 기능 데드엔드를 해결하기 위한 타겟팅된 핫픽스 방안을 담고 있습니다. 

검토 결과, 계획된 조치들은 기존 시스템 아키텍처에 불필요한 부작용을 야기하지 않으면서 기획자의 피드백을 100% 수용할 수 있는 무결한 설계로 판단됩니다. 특히 모달 토글 시점의 패스워드 강제 초기화 및 API 키 삭제 방어 유효성 검사 등 세부 보안 가드가 꼼꼼하게 설계되었습니다.

---

## 2. 세부 검토 결과 및 권고사항 (Detailed Review & Recommendations)

### [PRE-001] [우수] 가짜 사용자(Mock User) 자동 인증 완전 철거
- **검토 내용**: `RaptorWorkflow.tsx` 및 `AuthDashboard.tsx`에서 이전에 개발 편의를 위해 임시 삽입되었던 가짜 사용자 자동 주입 및 Step 1 스킵 바이패스 로직을 철거하는 계획입니다.
- **분석**: 진짜 Supabase Auth 세션의 `access_token`만을 타이트하게 연동하므로 인증 보안 누수가 완전히 해결됩니다.

### [PRE-002] [우수] 로그아웃 Zustand 상태 누수 리셋
- **검토 내용**: 로그아웃 시 Supabase 세션 아웃뿐만 아니라 Zustand Store 내의 핵심 비즈니스 데이터(`productData`, `analysis`, `finalAssets`, `manualAdditions`, `projectId` 등)를 완전히 초기화하는 `resetWorkflow()`를 연동합니다.
- **분석**: 이전 세션의 데이터가 로그아웃 후에도 남아 메모리에 캐싱되는 잔재 누수 현상을 완벽히 차단합니다.

### [PRE-003] [우수] 패스워드 폼 라이프사이클 보안 강화
- **검토 내용**: 로그인/가입 성공 및 실패, 로그아웃 시는 물론 **대시보드 모달이 토글(열기/닫기)되는 시점**에도 로컬 패스워드 상태를 `''`로 강제 리셋합니다.
- **분석**: 브라우저 메모리상에 비밀번호 텍스트가 필요 이상으로 유지되거나 노출되는 취약점을 차단하여 폼 보안성을 극대화합니다.

### [PRE-004] [우수] API 키 빈값 업데이트 차단 및 마스킹 플레이스홀더
- **검토 내용**: API 키 설정 창에서 인풋 필드를 빈칸으로 두고 업데이트를 수행할 때 기존 키가 존재할 경우 저장을 강제 반려하는 방어 유효성 검사를 작성하고, 마스킹된 플레이스홀더(`kie-***... (이미 설정됨)`)를 표기합니다.
- **분석**: 사용자의 실수로 기존에 설정된 HttpOnly 암호화 쿠키 내의 API Key를 공백으로 덮어써 날려버리는 사고를 방어하고, 마스킹 플레이스홀더를 통해 시각적 정합성을 제공합니다.

### [PRE-005] [우수] 프로필 버튼 레이아웃 고정 및 겹침 해결
- **검토 내용**: 프로필 버튼을 absolute에서 `fixed top-12 right-24 z-[100]`으로 교정합니다.
- **분석**: 랩터 로고 및 하단 콘텐츠 스크롤 시에도 최상단 z-index 레이어에서 안정적으로 동작하도록 포지셔닝을 정규화하여 겹침 결함을 해소합니다.

### [PRE-006] [우수] Step 3 렌더링 이동 데드엔드 차단
- **검토 내용**: 씬 스크립트만 생성되어 있다면 이미지 생성 완료 여부(`allImagesReady`)와 무관하게 다음 단계로 진입할 수 있도록 `canGoToStep4` 조건을 완화합니다.
- **분석**: AI 이미지 생성에 실패한 씬이 있더라도 사용자가 다음 렌더링 단계(Step 4)로 유연하게 넘어가 개별 재시도 또는 스틸컷 대체 등의 작업을 진행할 수 있도록 UX 흐름 단절을 완전히 방어합니다.
