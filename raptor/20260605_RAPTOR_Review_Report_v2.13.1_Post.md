## [Resolved] 해결 및 구현 완료 항목

### 1. 클라이언트 연결 끊김 감지 및 서버 자원 즉각 회수 (main.py)
*   **해결 내용**: 사용자가 브라우저 창을 닫거나 렌더링 도중 다른 페이지로 이동하여 클라이언트 접속이 해제되었을 때, 백엔드가 이를 인지하지 못하고 계속 KIE API를 폴링하거나 FFmpeg 작업을 이어가면서 요금 누수와 서버 자원 고갈이 발생하던 아키텍처적 위험을 해결했습니다.
*   **구현 상세**:
    1. FastAPI 라우터 파라미터로 `raw_request: Request`를 명시적으로 주입받았습니다.
    2. 비디오 생성 요청 스트리밍 엔진인 `render_stream` 및 `render_stream_test` 제너레이터 내부에서 핵심 3대 루프 돌입/순회마다 `await raw_request.is_disconnected()`를 실시간으로 점검하도록 보강했습니다.
    3. **KIE API 폴링 루프**, **비동기 태스크 완료 대기 루프**, **FFmpeg 최종 비디오 파일 인코딩 렌더링 루프**에서 연결 끊김이 감지되는 즉시 예외를 발생시키거나 루프를 탈출(`break`)하여 좀비 백그라운드 연산을 강제로 종료하고 자원을 즉각 회수합니다.

### 2. 렌더링 취소 버튼 및 크레딧 방어 UX 구현 (RaptorWorkflow.tsx)
*   **해결 내용**: 렌더링이 시작되면 완료될 때까지 사용자가 수동으로 작업을 멈출 수 없었던 결함을 해소하고, KIE API가 실제로 동작하여 크레딧(비용)이 나가기 시작하는 시점을 명확하게 구분하여 비용 누수를 사용자 수준에서 차단할 수 있게 했습니다.
*   **구현 상세**:
    1. `RaptorWorkflow.tsx` 내부에 `abortController` 상태와 이를 호출하는 `handleCancelRender` 취소 핸들러를 신설하고 SSE 연결(fetch) 시 해당 `signal`을 성공적으로 결합했습니다.
    2. 렌더링 진행 상황에 맞춰 `🛑 렌더링 취소` 버튼이 노출되며, KIE `taskId`가 발행되었는지 여부(즉, `finalAssets.script` 중 하나라도 `taskId` 속성을 보유하고 있는지)를 검사하여 UI를 동적으로 변경합니다.
    3. **크레딧 미소모 단계**: KIE Task ID가 없는 초기 연산 및 초기 대기 상태에는 **무채색 테두리 버튼**과 함께 `"취소 시 크레딧 미소모"` 문구를 노출합니다.
    4. **크레딧 소모/동영상 생성 단계**: KIE Task ID가 발행된 시점부터는 경고와 제어가 용이하도록 **파란색 버튼**으로 변환되며, `"현재 씬 생성 중 - 취소 시 대기 중인 다음 씬 비용 절약"`이라는 직관적인 가이드를 제공하여 불필요한 비용의 대량 지출을 조기에 방어할 수 있도록 설계했습니다.

### 3. [NEW-F 해결] 취소/단절 시 DB 내 태스크 상태 롤백(Failed) 복구 (main.py)
*   **해결 내용**: 백엔드에서 연결 중단 감지 시 연산만 정지하고 DB(`user_tasks_beta.json`) 상태를 갱신하지 않아 발생하던 사용자 대시보드 화면의 무한 로딩 리스크를 완벽하게 해결했습니다.
*   **구현 상세**: `main.py` 내의 `render_stream` 및 `render_stream_test` 제너레이터 내 비동기 씬 처리기 `process_scene_inner` 내부 전체를 `try-except asyncio.CancelledError` 및 `finally` 블록으로 래핑하여, 사용자가 취소하거나 단절될 때 DB 태스크 상태(`scene_task_id`)를 즉시 `failed`로 롤백하도록 복구 핸들러를 구현했습니다.

### 4. [RISK-003 해결] 크로스 플랫폼 폰트 경로 동적 매핑 (ffmpeg_worker.py)
*   **해결 내용**: 맑은고딕 폰트 경로 하드코딩(`C:/Windows/Fonts/malgun.ttf`)을 제거하고 리눅스/클라우드 배포 환경에서 비디오 합성 drawtext 필터가 폰트 파일 부재로 크래시되는 현상을 차단했습니다.
*   **구현 상세**: `os.name` 및 `platform` 감지를 통해 Windows 환경에서는 기존 경로를 유지하며, Linux/macOS 및 클라우드 배포판 환경에서는 리눅스 표준 폰트 후보 경로들(`/usr/share/fonts/...` 내 나눔고딕, DejaVuSans 등)을 자동 순회 탐색한 후 바인딩하도록 하였습니다. 만약 시스템에 폰트 파일이 전무할 시에도 크래시를 방지하기 위해 drawtext 필터의 제너릭 폰트명(`DejaVu Sans`)으로 유연하게 폴백하도록 예방 조치했습니다.

### 5. 병렬 scene_update 상태 업데이트 시 함수형 업데이트 적용 (RaptorWorkflow.tsx)
*   **해결 내용**: SSE 스트림에서 여러 씬들의 비디오 생성 완료 이벤트(`scene_update`)가 거의 동시에 발생할 때, 이전 상태를 기반으로 단순히 덮어쓰게(`setFinalAssets({ ...finalAssets, ... })`) 되면 상태 레이스 컨디션(Race Condition)으로 인해 특정 씬의 업데이트가 유실되던 문제를 제거했습니다.
*   **구현 상세**: `RaptorWorkflow.tsx` L.617 부근에서 상태 업데이트 함수 호출 시 이전 상태인 `prev` 인자를 콜백으로 받아 안전하게 수정하는 **함수형 업데이트 방식(`setFinalAssets((prev: any) => { ... })`)**으로 변경했습니다. 이로써 여러 비동기 스트림 데이터가 거의 동시에 병렬 수신되어도 누락 없이 모든 씬 상태가 독립적이고 확실하게 동기화됩니다.

### 6. 패키지 다운로드 파일 리소스 해제(URL.revokeObjectURL) 150ms 지연 처리
*   **해결 내용**: 렌더링 완료 후 결과 압축 패키지 다운로드 함수(`handleDownloadPackage`) 실행 시, 파일 저장 트리거 직후에 `URL.revokeObjectURL(blobUrl)`이 즉각 실행되어 일부 브라우저 환경에서 파일이 정상적으로 다운로드되지 않고 깨지거나 도중에 누락되던 버그를 정밀 해결했습니다.
*   **구현 상세**: `URL.revokeObjectURL`을 `setTimeout`을 통해 **150ms 지연 처리**하도록 조치하여 브라우저가 ZIP 다운로드 IO 스트림을 완전히 인지하고 프로세스를 실행하기에 충분한 안전 시간을 보장했습니다.

### 1. RISK-002 — KIE 모델 단가 분기 및 Storage FIFO 쿼터 한계
*   **리스크 내용**: 로컬 파일 시스템을 모방한 단일 JSON DB 기반 구조(`user_videos_beta.json`, `user_projects_beta.json`)가 유지되고 있습니다. 멀티 테넌트(다중 사용자) 클라우드 환경으로 가면서, 여러 사용자가 동시에 대량의 영상 생성을 시작하면 I/O 경합이 유발되고 FIFO(최대 50개) 정제 작업 시 다른 사용자의 리소스를 침범하거나 성능 지연을 초래할 수 있으므로 속히 PostgreSQL 등의 프로덕션 관계형 DB로의 마이그레이션이 수립되어야 합니다.
