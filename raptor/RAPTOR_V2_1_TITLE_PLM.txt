[RAPTOR V2.1 TITLE PLM]

1. 제목 생성 목표
- 스크롤을 멈추게 한다
- 핵심 효용을 빠르게 전달한다
- 호기심을 유도한다
- 과장 없이 관심을 끈다

2. 제목 길이 규칙
- 권장 길이: 18자 ~ 32자

3. 제목 구조 원칙
- 문제 → 해결
- 변화 → 결과
- 상황 → 해결
- 호기심 → 확인

4. 제목 생성 우선 요소
- 사용자 문제
- 제품 핵심 장점
- 전후 변화
- 사용 상황
- 타겟 고객

5. 제목 각도 유형
- 문제 해결형
- 손해 회피형
- 전후 비교형
- 호기심형
- 효율 상승형
- 공감형
- 추천형
- 간단 해결형

6. 제목 예시
문제 해결형
- 정리 안 되면 이거 하나면 끝
- 책상 정리 고민이면 이거 보세요

손해 회피형
- 이거 모르고 계속 고생했습니다
- 이거 안 쓰면 계속 불편합니다

전후 비교형
- 이거 쓰기 전이랑 완전히 다릅니다
- 이거 하나로 책상이 달라집니다

호기심형
- 왜 이제 알았을까요
- 이거 생각보다 편합니다

효율 상승형
- 이거 하나로 일이 빨라집니다
- 작업 효율 올리는 아이템

공감형
- 직장인이라면 이거 보세요
- 자취생이면 이거 필요합니다

7. 카테고리별 제목 예시
생활용품
- 생활 꿀템 하나 찾았습니다
- 이거 하나로 집이 달라집니다

전자기기
- 이거 하나로 작업 환경 달라집니다
- 업무 효율 올리는 아이템

주방용품
- 요리할 때 이거 하나면 편합니다
- 주방 필수템 하나 찾았습니다

뷰티
- 이거 써보면 왜 인기인지 압니다
- 관리 루틴 편해지는 아이템

자동차용품
- 차에 두면 은근 편합니다
- 운전자라면 한번 보세요

사무용품
- 직장인 책상 필수템
- 책상 위 정리템 찾았습니다

8. 제목 출력 규칙
- 기본 5개 생성
- 각도 겹치지 않게 만든다
- 가장 강한 클릭형 1개 포함

9. 썸네일 문구 규칙
- 3~6글자 중심
- 짧고 강하게
- 의미 즉시 이해 가능
- 후킹 감정 또는 핵심 효용 반영

10. 썸네일 문구 예시
- 일속도2배
- 이거필수
- 진짜편함
- 책상혁명
- 공간확보
- 시간절약
- 정리끝
- 왜이제앎
