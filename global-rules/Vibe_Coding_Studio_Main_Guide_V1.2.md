📘 Vibe Coding Studio Main Guide V2.1
(Context-Preserved & Scenario-First Framework, 2026.05)

0. What this file is
이 문서는 모든 Auto Studio의 공통 프레임워크이자 에이전트 행동 강령입니다.
V2.1은 **'사용자 시나리오 맥락 유지 및 임기응변 금지'** 원칙을 시스템에 강제합니다.

--------------------------------------------------------------------------------
1. 프로젝트 목적
LLM을 활용해 자연어 → 코드/자동화로 이어지는 직감적 창작·개발을 표준화한다. 
V2.1은 기술적 구현보다 **사용자 경험 시나리오(User Journey)**의 연속성을 최우선 가치로 삼는다.

--------------------------------------------------------------------------------
2. 3-문서 체계 (PRD / MD / RULES)
PRD: 목적·사용자·기능·KPI (시나리오 맵핑 포함)
MD: Step-by-Step 사용자 워크플로우 설계 / 프롬프트·모듈 구조
RULES: 협업·버전·Context Preservation 규칙
이 3문서는 AI-Human 협업의 언어적 계약이며, 컨펌 없이는 다음 단계로 진행 불가하다.

--------------------------------------------------------------------------------
3. Auto-Track (6단계) – 사용자 맥락 중심
1) PRD (자연어 요구정의) → 산출물: PRD_Vx.md
2) **User Journey Mapping & Human Confirmation (신설)**: 전체 시나리오 흐름 확정
3) MD (워크플로우 & 흐름 설계) → 산출물: MD_Vx.md
4) RULES (품질·맥락 유지 규칙) → 산출물: RULES_Vx.md
5) AI Coding/Creation (TDD 기반 생성) → 산출물: 프로토타입
6) Automation (연결·배포·리포트) → 산출물: 워크플로/리포트

--------------------------------------------------------------------------------
4. 운영 프로세스 (SOP)
아이디어 → PRD → **시나리오 컨펌(HIL)** → MD → RULES → 생성 → 자동화 → 리포트

flowchart TD
    A[Idea]-->B[PRD]
    B-->C{User Journey Confirm}
    C-- Approved -->D[MD/RULES]
    C-- Rejected -->B
    D-->E[Create/Code]
    E-->F[Automation]
    F-->G[Report/Improve]

--------------------------------------------------------------------------------
5. 핵심 원칙: Context & Structural Integrity
- **임기응변(땜빵) 금지:** 특정 에러를 해결하기 위해 전체 시나리오 맥락이나 아키텍처를 훼손하는 수정을 절대 금지한다.
- **맥락 유지 의무:** '문맥 다이어트'를 이유로 비즈니스 핵심 플로우를 누락하지 않는다.
- **선 설계 후 코딩:** 모든 코딩은 인간에게 승인받은 사용자 시나리오 흐름 내에서만 이루어진다.

--------------------------------------------------------------------------------
6. 보안·정책
API 키 보호, 개인정보 미포함, 플랫폼 정책(YouTube/TikTok Shop 등) 준수 필수.

--------------------------------------------------------------------------------
7. 핵심 메시지
코드는 쓰는 것이 아니라 대화로 태어나고, 사용자 시나리오의 맥락 안에서만 생존한다.
