# Product Specification: YT-Comment Insight Engine V2

## 1. 개요
1단계(Collection)에서 수집된 유튜브 댓글 데이터를 정제, 분석하여 하이브리드(로컬 NLP + LLM) 방식으로 인사이트를 도출하는 분석 엔진.

## 2. 주요 기능 명세 (Functional Specs)

### [F-01] 데이터 전처리 및 익명화 (Anonymization)
- **입력:** `1_collection/data/{video_id}/*.csv`
- **처리:** 
  - 댓글 작성자 ID(`authorChannelId`)를 SHA-256으로 해싱하여 익명화.
  - 이모지, 특수문자, 불용어 제거 및 텍스트 정규화.
- **출력:** SQLite DB (`analysis_cache.db`)에 저장.

### [F-02] 하이브리드 분석 파이프라인 (Hybrid Pipeline)
- **로컬 분석 (Local NLP):** 
  - **감성 분석:** KcBERT-Small 모델을 사용하여 긍정/부정/중립 판별.
  - **유해성 탐지:** 스팸, 혐오 표현 필터링.
  - **토픽 모델링:** BERTopic을 활용한 주제 군집화.
- **LLM 인사이트 (Selective LLM):** 
  - 로컬에서 추출된 상위 5% 핵심 댓글 및 토픽 요약을 기반으로 최종 리포트 생성.
  - 비용 방어를 위해 전체 데이터의 5% 이내로 토큰 전송 제한.

### [F-03] 여론 왜곡 및 SNA 분석
- **네트워크 맵:** 댓글-답글 관계 및 작성자 활동 빈도를 기반으로 한 클러스터 분석.
- **왜곡 탐지:** 특정 시간대 반복 댓글, 소수 계정의 독점적 의견 게시 패턴 탐지.

### [F-04] 단일 화면 대시보드 (SPA UI)
- **상태 표시:** 분석 진행률, 로컬 모델 로딩 상태 실시간 표시.
- **시각화:** 
  - AISAS/의도 분류 차트.
  - 감성 분포 도넛 차트.
  - 시계열 활동량 그래프.
- **CSV 내보내기:** 연구용 정제 데이터 다운로드.

## 3. 예외 처리 및 결함 허용 (Fault Tolerance)
- `analysis_checkpoint.json`을 사용하여 분석 도중 중단 시 마지막 완료 시점부터 재개.
- API 키 누락 또는 쿼터 초과 시 로컬 분석 결과만으로 대시보드 우선 구성.

## 4. 테스트 포인트 (For Agent 3)
- 비정상적 형식의 CSV 파일 입력 시 예외 처리 확인.
- 해시 처리된 ID가 원본과 1:1 매칭되는지(중복 방지) 확인.
- 로컬 모델 추론 속도 및 메모리 점유율 임계치 확인.
