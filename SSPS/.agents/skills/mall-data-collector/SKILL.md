# Mall Data Collector Skill

## 역할 (Role)
이 스킬은 SSPS 시스템의 **커넥터 계층**을 담당합니다. 입력된 분야(domain)를 파악하여 가장 신뢰도 높은 데이터 소스부터 우선 수집하고, 실패 시 회로 차단기(Circuit Breaker)를 통해 대체 소스로 폴백합니다.

## 데이터 소스 우선순위 규칙

### 1. 뷰티 카테고리 (Beauty Category)
- **트리거 키워드**: 스킨케어, 색조, 헤어, 뷰티, 화장품, 립스틱, 마스크팩, 선크림, 앰플, 세럼, 토너, 에센스
- **수집 순서**: 올리브영 랭킹 → 네이버 쇼핑검색 → 네이버 데이터랩
- **올리브영**: 카테고리별 Top 30 랭킹, 리뷰 수, 평점
- **네이버 쇼핑**: 브랜드 키워드 검색, 가격 범위

### 2. 저가/생활용품 카테고리 (Low-Price Category)
- **트리거 키워드**: 수납, 청소, 생활, 주방, 욕실, 정리, 다이소, 1000원, 저가, 생활용품
- **가격 필터**: 1,000원 ~ 30,000원
- **수집 순서**: 다이소 랭킹 → 네이버 쇼핑검색 → 네이버 데이터랩
- **다이소**: 카테고리별 베스트 랭킹, 신상품

### 3. 시즌성/트렌드 카테고리 (Seasonal Category)
- **트리거 키워드**: 장마, 여름, 겨울, 환절기, 벚꽃, 명절, 캠핑, 피크닉
- **수집 순서**: 네이버 데이터랩 → 네이버 쇼핑검색 → 올리브영/다이소
- **네이버 데이터랩**: 최대 1년치 검색 클릭 추이, 계절성 분석

### 4. 일반 카테고리 (General Category)
- **수집 순서**: 네이버 쇼핑검색 → 네이버 데이터랩 → 올리브영

## 수집 실행 절차

```
1. 입력 텍스트에서 카테고리 타입 감지
2. 해당 카테고리에 맞는 커넥터를 우선순위 순으로 호출
3. 각 커넥터에 타임아웃 15초 설정
4. 실패 시 Circuit Breaker가 자동으로 Mock 데이터로 폴백
5. 수집된 원시 데이터를 표준 포맷으로 정규화하여 반환
```

## 반환 데이터 표준 포맷
```json
{
  "source": "oliveyoung | daiso | naver_shopping | naver_datalab",
  "status": "OK | CIRCUIT_OPEN | MOCK",
  "items": [
    {
      "name": "상품명",
      "price": 12000,
      "rank": 1,
      "review_count": 5432,
      "rating": 4.8,
      "trend_score": 85.0,
      "image_url": "https://..."
    }
  ]
}
```

## 에러 처리 규칙
- **HTTP 403/429**: 즉시 Circuit Breaker 트립, Mock 데이터 사용
- **타임아웃**: 15초 초과 시 부분 결과 반환
- **파싱 오류**: 경고 로깅 후 다음 소스로 전환
- **부분 실패**: 1개 소스 실패 시에도 나머지 소스로 스코어링 진행
